err()

AdminErrors.php : 64

Metodo dedicato alla gestione interna degli errori

public err(array<string|int, mixed> $args[, bool $short = false ][, array<string|int, mixed> $trace = [] ]) : string

Questo metodo è progettato per gestire gli errori interni al plugin in modo centralizzato. Utilizza il sistema di logging di WordPress per registrare gli errori in un file di log specifico del plugin e aggiunge gli errori a un'istanza di WP_Error per un'ulteriore gestione.

Il metodo raccoglie informazioni dettagliate sull'errore, come la classe e il metodo che lo hanno generato, il file e la linea di codice in cui si è verificato, e un messaggio personalizzato. Queste informazioni sono utili per il debug e la manutenzione del codice.

ATTENZIONE ! Se viene chiamata da un wrapper, passare il paramentro $trace valorizzato in questo modo:

// prende due livelli, quello presente e il chiamante e ignora gli argomenti delle funzioni per alleggerire il dataset
debug_backtrace(DEBUG_BACKTRACE_IGNORE_ARGS, 2)
Parameters
$args : array<string|int, mixed>

Un array associativo contenente i dettagli dell'errore:

  • 'code' (string): Codice identificativo dell'errore (default: 'SEA_ERR').
  • 'message' (string|array): Messaggio dell'errore. Se è un array, può contenere:
    • 'fix' (string): Parte fissa del messaggio.
    • 'var' (string): Variabile aggiuntiva da includere nel messaggio.
$short : bool = false

restituisce una stringa con solo il messaggio, senza tutte le info

$trace : array<string|int, mixed> = []

se viene passato prende questo backtrace, se no prende i dati del chiamante

Return values
string
