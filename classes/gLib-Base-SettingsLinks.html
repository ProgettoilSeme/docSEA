array_json()

BaseController.php : 828

Recupera dal DB array di valori da un campo JSON o serializzato

public array_json(string $input[, string|null &$err = '' ]) : array<string|int, mixed>

Questo speciale metodo occorre per visualizzare i dati presi da DB e in particolare in backend, quando abbiamo nel DB dei dati serialzzati che sono stati letti dalle get data delle classi specializzate, che si aspettano di restituire al frontend il dato in JSON, così come l'hanno ricevuto

Il processo di sanitizzazione che gestisce la sicurezza è relativamente complesso e consta di tre passaggi sia in INPUT che in OUTPUT.

  • Cosa succede nel flusso fuori da questa funzione:

In INPUT, da frontend, quando viene ricevuto il dato in formato JSON sono eseguiti in serie:

  • stripslashes
  • json_decode
  • serialize

Quando viene restituito il JSON al frontend, viene fatta l'operazione iversa:

  • json_encode
  • addslashes
  • esc_attr
  • Cosa succede nel flusso quando è chiamata questa funzione:

In vari casi può essere utile che il dato sia intercettato e restituito come semplice vettore.

In questi casi vanno eseguti i passi:

  • stripslashes: opzionale, se ci sono slash di troppo
  • html_entity_decode: opzionale, se ci sono caratteri HTM
  • json_decode/unserialize: esegue la decodifica a seconda della stringa passata
Parameters
$input : string

stringa in input contenente il json o il dato serializzato

$err : string|null = ''

eventuale errore se non riesce a riconoscere il formato

Return values
array<string|int, mixed> —

restituisce un array corrispondente alla stringa

get_permission_provider()

BaseController.php : 619

Metodo per il recupero dei permessi da fornitore SEA di un certo WP_User

public get_permission_provider(WP_User $WP_User[, array<string|int, mixed> &$SEA_caps = [] ][, string|bool &$SEA_role = false ][, mixed &$err = '' ]) : bool|string

La logica è semplice: a seconda del grado di importanza di un dato ruolo/caps, si "trattiene" quello minimo sufficiente per determinare se i permessi ci sono, quindi si parte dai ruoli amministrativi con pieni permessi, e poi si discende per arrivare a quelli di un generico utente SEA che può o non può averli a seconda del tipo di profilo.

Questo metodo preleva dalla classe AdminUsers i ruoli predefiniti SEA e considera insieme a questi quelli amministrativi WP.

I permessi sono definiti sia dai ruoli che dalle caps (capability) associate ai ruoli. Tuttavia per definire se un utente è abilitato contano di più le caps. * Questo metodo tenta di usare l'option:

BaseController::SEA_PLUGIN_ID . '_customer_users';

dove sono salvaguardati per comodità tutti i profili SEA in modo da tenerli distinti da quelli WP, e nel caso non dovesse trovare l'utente tra quelli registrati nella option, genera un errore e poi tenta di recuperare i caps direttamente dal profilo. I caps sono legati in via fissa a un preciso ruolo, quindi se ad esempio l'utente WP ha ruolo 'promoter' che è proprietario SEA, allora avremo per forza il caps 'view_profit' => 1, cioè a true.

Parameters
$WP_User : WP_User
$SEA_caps : array<string|int, mixed> = []

Restituisce le caps trovate

$SEA_role : string|bool = false
$err : mixed = ''

Restituisce un errore

Return values
bool|string

err()

BaseController.php : 169

Metodo dedicato alla gestione interna degli errori

protected err(array<string|int, mixed> $args) : string

E' un wrapping di AdminErrors::err(...)

Questo metodo è progettato per gestire gli errori interni al plugin in modo centralizzato. Utilizza il sistema di logging di WordPress per registrare gli errori in un file di log specifico del plugin e aggiunge gli errori a un'istanza di WP_Error per un'ulteriore gestione.

Il metodo raccoglie informazioni dettagliate sull'errore, come la classe e il metodo che lo hanno generato, il file e la linea di codice in cui si è verificato, e un messaggio personalizzato. Queste informazioni sono utili per il debug e la manutenzione del codice.

Parameters
$args : array<string|int, mixed>

Un array associativo contenente i dettagli dell'errore:

  • 'code' (string): Codice identificativo dell'errore (default: 'SEA_ERR').
  • 'message' (string|array): Messaggio dell'errore. Se è un array, può contenere:
    • 'fix' (string): Parte fissa del messaggio.
    • 'var' (string): Variabile aggiuntiva da includere nel messaggio.
Return values
string
