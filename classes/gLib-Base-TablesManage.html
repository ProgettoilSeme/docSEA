TablesManage.php : 41

Classe TablesManage

Questa classe raccoglie vari metodi di uso generico per la gestione dei dati organizzati per tabelle (cioè matrici o array numerici di array associativi) dove un record corrisponde a una riga e un dato a una colonna.

Esempio:

ID Nome Attivo
1 Mario Sì
2 Luisa No
3 Paolo Sì

Il primo record è un array PHP:

$record = [ 'ID' => 1, 'Nome' => 'Mario', 'Attivo' => 'Si' ]

Mentre la matrice sarà la lista di array:

$matrix = [
     0 => [ 'ID' => 1, 'Nome' => 'Mario', 'Attivo' => 'Si' ],
     1 => [ 'ID' => 2, 'Nome' => 'Luisa', 'Attivo' => 'Np' ],
     2 => [ 'ID' => 3, 'Nome' => 'Paolo', 'Attivo' => 'Si' ]
]

Per questo genere di strutture che possono essere restituite da unq QUERY non esiste nativamente in PHP un metodo per la rimozione di una colonna della matrice o la possibilità di "rinominare" la colonna e quindi la chiave associativa di tutti i valori di una data colonna.

public mapping_out(array<string|int, mixed> $dataset, array<string|int, mixed> $mapping_out) : array<string|int, mixed>

Accetta un dataset o recorset, cioè un insieme di record, oppure un singolo record che contiene direttamente i valori da gestire.

Nello scenario classico, viene ricevuto un recordset dal DB e $mapping_out individua le colonne le cui chiavi corrispondenti ai nomi delle colonne DB vanno rimappate in quanto hanno come valore la chiave corretta, per ciò mi aspetto che TUTTE le chiavi da rimappare siano nel vettore dato.

add()

TablesManage.php : 92

Metodo dedicato alla gestione interna degli errori

protected add([string $code = '' ][, string $fix_message = '' ][, string $var_message = '' ]) : string

E' un wrapping di AdminErrors::add(...)

Questo metodo è progettato per gestire gli errori interni al plugin in modo centralizzato. Utilizza il sistema di logging di WordPress per registrare gli errori in un file di log specifico del plugin e aggiunge gli errori a un'istanza di WP_Error per un'ulteriore gestione.

Il metodo raccoglie informazioni dettagliate sull'errore, come la classe e il metodo che lo hanno generato, il file e la linea di codice in cui si è verificato, e un messaggio personalizzato. Queste informazioni sono utili per il debug e la manutenzione del codice.
