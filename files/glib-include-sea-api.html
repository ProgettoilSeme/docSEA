SEA_API.php : 55

Funzione di wrapping per restituire l'istanza corretta dall'interfaccia.

SEA_instance() : iSEA_API

GET o INIT Se quello attuale è un Utente SEA (ruolo) o Amministratore ed è la prima chiamata, pertanto non ci sono fornitori associati per questo utente, crea un nuovo record nella tabella WP_SEA_provviders associandolo e lo restituisce 'vuoto'. Altrimenti, se c'è almeno un fornitore associato, restituisce i dati che trova su DB.

HEADER PHP (required) per TEMA Viene incluso automaticamente nel tema dalla LibraryFrontEnd.

L'interfaccia SEA_API espone i metodi SET e GET. È implementata sempre dalla classe LibraryFrontEnd.

ATTENZIONE: questa funzione CREA LA TABELLA se non esiste, quindi se è la prima chiamata dopo l'installazione del plugin. Successivamente tutte le modifiche verranno effettuate modificando il record esistente. Tutti gli stati di errore sono gestiti internamente!!!

Esempio di utilizzo:

// Registra (inizializzando) il provider se non esiste, oppure il provider registrato se lo trova
// per registrare/restituire il provider, usa l'ID dell'utenza corrente
//= PROVIDER
global $current_user;
$prd_status = 0; /// restituisce il record provider dell'utente corrente (-1 error/inconsistente) (-2 duplicato)
$provider = SEA_get_provider($prd_status);

//= LOCATION
$idPippo = 0;
$locationError = '';
$locationNew = $obj_SEA_API->instance()->SEA_get_array_json_prv('locations', $idPippo, $locationError);

//= PRODUCT
$prv_status = 0; /// restituisce il record provider dell'utente corrente (-1 error) (-2 duplicato)
$prv_id = 0;
$prd_err = '';
$product = SEA_get_product($prv_status, $prv_id, (int)esc_html(get_query_var("edit")), $prd_err);
