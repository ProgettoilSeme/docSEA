Servizi principali del plugin

  • Dashboard intera per l’attivazione dei servizi SEA
  • Amministrazione dedicata agli utenti SEA
  • Amministrazione dei profili dedicati per gli utenti SEA (con impostazioni automatiche di default)
  • Amministrazione dei Fornitori
  • Amministrazione dei Prodotti
  • Gestione registrazione
  • (toDo) Configurazione SMTP per invio email

Come estendere i servizi

  1. Copia la classe AdminMyClass e segui le istruzioni nei commenti
  2. Aggiungi il metodi di callback nella sezione di libreria dedicata copiando il file MyClassCallbacks.
  3. Aggiungi l’ID di attivazione del servizio nella classe BaseController.
  4. Dichiara la classe nella Init e “attivare” il servizio dal pannello di amministrazione del plugin.

Per maggiori dettagli, consulta la documentazione delle singole classi e metodi.

Logica Generale

Il plugin gestisce sia il backend che il frontend per l’amministrazione di serivizi dedicati al terzo settore. Si tratta di gestire i dati di un marketplace senza scopo di lucro, dove i fornitori sono soggetti individuati da alcuni un gruppo di quattro profili proprietari: - fornitore senza scopo di lucro - fornitore - utente senza scopo di lucro - utente

Il sito non è pensato per gestire transazioni economiche, dato che il valore scambiato è l’accordo tra le parti aventi diffenti interessi. Ad esempio un utente del terzo settore potrebbe avere necessità di suppellettili che un altro utente può mettere a disposizione.

La complessità amministrativa è quindi discreta: oltre a gestire i nuovi profili, vanno gestiti i dati dei fornitori e dei loro prodotti.

Il plugin fornisce anche un supporto diretto e indiretto al frontend, secondo una logica standardizzata: - nel momento in cui accedo a una pagina per inizializzarla, chiamo una funzione PHP che prende i dati da DB - se la request proviene da backend, viene gestita dalle Session API di WordPress tramite POST - se la request proviene da frontend ed è una modifica, viene gestita da chiamate AJAX sempre tramite POST - se la request proviene da frontend è per un recordset visualizzato a blocchi, viene gestita con le REST API

SEA_instance() : iSEA_API

GET o INIT Se quello attuale è un Utente SEA (ruolo) o Amministratore ed è la prima chiamata, pertanto non ci sono fornitori associati per questo utente, crea un nuovo record nella tabella WP_SEA_provviders associandolo e lo restituisce 'vuoto'. Altrimenti, se c'è almeno un fornitore associato, restituisce i dati che trova su DB.

HEADER PHP (required) per TEMA Viene incluso automaticamente nel tema dalla LibraryFrontEnd.

L'interfaccia SEA_API espone i metodi SET e GET. È implementata sempre dalla classe LibraryFrontEnd.

ATTENZIONE: questa funzione CREA LA TABELLA se non esiste, quindi se è la prima chiamata dopo l'installazione del plugin. Successivamente tutte le modifiche verranno effettuate modificando il record esistente. Tutti gli stati di errore sono gestiti internamente!!!

Esempio di utilizzo:

// Registra (inizializzando) il provider se non esiste, oppure il provider registrato se lo trova
// per registrare/restituire il provider, usa l'ID dell'utenza corrente
//= PROVIDER
global $current_user;
$prd_status = 0; /// restituisce il record provider dell'utente corrente (-1 error/inconsistente) (-2 duplicato)
$provider = SEA_get_provider($prd_status);

//= LOCATION
$idPippo = 0;
$locationError = '';
$locationNew = $obj_SEA_API->instance()->SEA_get_array_json_prv('locations', $idPippo, $locationError);

//= PRODUCT
$prv_status = 0; /// restituisce il record provider dell'utente corrente (-1 error) (-2 duplicato)
$prv_id = 0;
$prd_err = '';
$product = SEA_get_product($prv_status, $prv_id, (int)esc_html(get_query_var("edit")), $prd_err);
